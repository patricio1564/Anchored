## What We Completed

### 1. Fixed StreakManager Crash in Lesson Flow
**Problem:** Tapping any lesson in the Topics tab crashed with:
> Fatal error: No Observable object of type StreakManager found.

**Root cause:** `LessonView` uses `@Environment(StreakManager.self)` but the Topics navigation stack never injected a `StreakManager`. `HomeView` and `ProfileView` each created their own local instance, but the environment was never populated for the Learn tab path.

**Fix — `Anchored/App/MainTabView.swift`:**
- Added `@EnvironmentObject private var authManager: AuthManager`
- Added `@Environment(\.modelContext) private var modelContext`
- Added `@State private var streakManager: StreakManager?`
- `var body` wraps the TabView in a `Group`: if `streakManager` is ready, injects it via `.environment(streakManager)`; otherwise renders the tabs with a `.task` that creates and stores the manager
- The actual TabView moved to `private var tabContent: some View`
- Uses `authManager.currentUserId` (not `userId` — the property is `currentUserId` on `AuthManager`)

**Architecture note:** `HomeView` and `ProfileView` still create their own local `StreakManager` instances via `@State`. This is intentional — they existed before this fix and drive their own tab's UI. The environment-injected manager from `MainTabView` exists solely to satisfy `LessonView`'s `@Environment(StreakManager.self)` requirement. Both instances write to the same SwiftData store so data is consistent.

## Decisions & Conventions to Remember

### AuthManager
- The published property for the signed-in user ID is `currentUserId: String?` (computed from the `state` enum), NOT `userId`
- `AuthManager.state` is an enum: `.signedOut` / `.signedIn(userId:, displayName:)`

### StreakManager
- `@Observable` (not `ObservableObject`) — read with `@Environment(StreakManager.self)`, not `@EnvironmentObject`
- Requires `ModelContext` + `userId` at init — cannot be created before SwiftData is available
- Views that need it from the environment: `LessonView` — now satisfied by `MainTabView`
- Views that create their own local copy: `HomeView`, `ProfileView` — leave these alone, they work

### BibleVerse: Identifiable
- `extension BibleVerse: @retroactive Identifiable` is declared in `BiblePassageView.swift`
- The `@retroactive` attribute is required because `BibleVerse` is defined in a different module
- Do not declare this anywhere else — it will cause a redeclaration error

### NoteComposeSheet
- Moved to `BiblePassageView.swift` (was previously in `BibleView.swift`)
- It is a `struct` (not private) so `BiblePassageView` can reference it

### Chapter Tile Rendering
- Do NOT use `aspectRatio(1:1)` + `.anchoredStyle(...)` inside `NavigationLink` labels in a `LazyVGrid` — this causes text truncation for many tiles
- Use fixed height: `frame(maxWidth: .infinity, minHeight: 44, maxHeight: 44)` + `.font(.system(size: 15, weight: .medium))` + `.lineLimit(1)`

### XcodeGen
- `project.yml` is at `/Users/patrickray/Downloads/Anchored-merged/project.yml`
- Always run `xcodegen generate` from `Anchored-merged/` (the directory containing `project.yml`)
- Must re-run after adding, renaming, or deleting any Swift files
